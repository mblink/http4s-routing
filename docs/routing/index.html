Routing

Once you've built your Routes, they can be used to route http4s requests to specific application logic. Each route contains enough information to match it against a request and extract the relevant parameters from the request's path and query string, so all you need to do is specify the route-specific handling logic. Assuming you have the test routes:

import cats.instances.int._
import cats.instances.string._
import org.http4s.dsl.io._
import org.http4s.routing._

val Login = GET / "login"
// Login: Route[org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, Unit]{type PathParams = Unit;type QueryParams = Unit}{type Method = org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody} = /login
val Hello = GET / "hello" / ("name" -> StringVar)
// Hello: Route[org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, (Unit, String)]{type PathParams = (Unit, String);type QueryParams = Unit}{type Method = org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody} = /hello/<name: String>
val BlogPost = GET / "post" / ("slug" -> StringVar) :? queryParam[Int]("id")
// BlogPost: Route[org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, ((Unit, String), Int)]{type PathParams = (Unit, String);type QueryParams = (Unit, Int)}{type Method = org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody} = /post/<slug: String>?<id: Int>

Then you can specify the handling logic for a given route by calling handle.with.logic_, which takes a function of the type:

// where `P` is a tuple of the route's parameter types
Request[F] => P => F[Response[F]]

Here's how you could handle the routes above:

import cats.effect.IO
import org.http4s.Request

val handledLogin = Login.handle.with_[IO](_ => _ => Ok("Login page"))
// handledLogin: Handled.Aux[IO, org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, Unit, Unit, Route[org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, Unit]{type PathParams = Unit;type QueryParams = Unit}{type Method = org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody}] = org.http4s.routing.Handled$Ops$RouteHandlerOps$With$$anon$1@10c953cc
val handledHello = Hello.handle.with_[IO](_ => name => Ok(s"Hello, $name"))
// handledHello: Handled.Aux[IO, org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, String, (Unit, String), Route[org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, (Unit, String)]{type PathParams = (Unit, String);type QueryParams = Unit}{type Method = org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody}] = org.http4s.routing.Handled$Ops$RouteHandlerOps$With$$anon$1@7fcd42a2
val handledBlogPost = BlogPost.handle.with_((req: Request[IO]) => { case (slug, id) =>
  Ok(s"Blog post with id: $id, slug: $slug found at ${req.uri}")
})
// handledBlogPost: Handled.Aux[IO, org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, (String, Int), ((Unit, String), Int), Route[org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, ((Unit, String), Int)]{type PathParams = (Unit, String);type QueryParams = (Unit, Int)}{type Method = org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody}] = org.http4s.routing.Handled$Ops$RouteHandlerOps$With$$anon$1@62ee3366

And finally, you can compose your handled routes into a service by passing them to Route.httpRoutes:

import cats.effect.IO
import org.http4s.HttpRoutes

val service1: HttpRoutes[IO] = Route.httpRoutes(
  handledLogin,
  handledHello,
  handledBlogPost
)
// service1: HttpRoutes[IO] = Kleisli(
//   org.http4s.Http$$$Lambda$16574/1507257495@465ffffe
// )

If you prefer, you can build an HttpRoutes manually by matching on your Routes:

val service2: HttpRoutes[IO] = HttpRoutes.of {
  case Login(_) => Ok("Login page")
  case Hello(name) => Ok(s"Hello, $name")
  case req @ BlogPost(slug, id) =>
    Ok(s"Blog post with id: $id, slug: $slug found at ${req.uri}")
}
// service2: HttpRoutes[IO] = Kleisli(
//   org.http4s.HttpRoutes$$$Lambda$16575/682619860@1c5293ca
// )

You can confirm that routes are matched correctly by passing some test requests to the service:

import org.http4s.Request

def testRoute(service: HttpRoutes[IO], call: Call) =
  service
    .run(Request[IO](method = call.route.method, uri = call.uri))
    .value
    .unsafeRunSync
    .get
    .as[String]
    .unsafeRunSync

testRoute(service1, Login())
// res0: String = "Login page"
testRoute(service1, Hello("world"))
// res1: String = "Hello, world"
testRoute(service1, BlogPost("my-slug", 1))
// res2: String = "Blog post with id: 1, slug: my-slug found at /post/my-slug?id=1"

testRoute(service2, Login())
// res3: String = "Login page"
testRoute(service2, Hello("world"))
// res4: String = "Hello, world"
testRoute(service2, BlogPost("my-slug", 1))
// res5: String = "Blog post with id: 1, slug: my-slug found at /post/my-slug?id=1"

You can also check that requests matching none of your routes are not handled by the service:

import org.http4s.{Method, Uri}

def unhandled(method: Method, path: String) =
  service1
    .run(Request[IO](method = method, uri = Uri(path = path)))
    .value
    .unsafeRunSync

unhandled(GET, "/fake")
// res6: Option[org.http4s.Response[IO]] = None

// Not handled by `Hello` because the method doesn't match
unhandled(POST, "/hello/world")
// res7: Option[org.http4s.Response[IO]] = None
