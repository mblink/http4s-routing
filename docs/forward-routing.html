Forward Routing

Once you've built your Routes, you can compose them into an http4s service, where they'll be used to match a request coming into your application, extract the relevant parameters from it, and perform route-specific logic. First let's define our routes:

import cats.instances.int._
import cats.instances.string._
import org.http4s.dsl.io._
import org.http4s.routing._

val Login = GET / "login"
// Login: Route[org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, Unit]{type PathParams = Unit;type QueryStringParams = Unit}{type Method = org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody} = /login
val Hello = GET / "hello" / ("name" -> StringVar)
// Hello: Route[org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, (Unit, String)]{type PathParams = (Unit, String);type QueryStringParams = Unit}{type Method = org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody} = /hello/<name: String>
val BlogPost = GET / "post" / ("slug" -> StringVar) :? queryParam[Int]("id")
// BlogPost: Route[org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody, ((Unit, String), Int)]{type PathParams = (Unit, String);type QueryStringParams = (Unit, Int)}{type Method = org.http4s.Method with org.http4s.Method.Semantics.Safe with org.http4s.Method.PermitsBody} = /post/<slug: String>?<id: Int>

You can specify the handling logic for a given route by calling .handled on it. handled takes a function from the route's parameters to an http4s response — F[Response[F]].

val handledLogin = Login.handled(_ => Ok("Login page"))
// handledLogin: Route.Handled[cats.effect.IO, _1.route.type] = org.http4s.RoutingLP$RouteOps$$anon$1@3a670b35
val handledHello = Hello.handled(name => Ok(s"Hello, $name"))
// handledHello: Route.Handled[cats.effect.IO, _1.route.type] = org.http4s.RoutingLP$RouteOps$$anon$1@7c9325ad
val handledBlogPost = BlogPost.handled { case (slug, id) => Ok(s"Blog post with id: $id, slug: $slug") }
// handledBlogPost: Route.Handled[cats.effect.IO, _1.route.type] = org.http4s.RoutingLP$RouteOps$$anon$1@75b048a6

And finally, you can compose our handled routes into a service by passing them to Route.httpRoutes:

import cats.effect.IO
import org.http4s.HttpRoutes

val service: HttpRoutes[IO] = Route.httpRoutes(
  handledLogin,
  handledHello,
  handledBlogPost
)
// service: HttpRoutes[IO] = Kleisli(
//   org.http4s.Http$$$Lambda$14608/1575199479@287a36dc
// )

You can confirm that routes are matched correctly by passing some test requests to the service:

import org.http4s.Request

def testRoute(call: Call) =
  service
    .run(Request[IO](method = call.route.method, uri = call.uri))
    .value
    .unsafeRunSync
    .get
    .as[String]
    .unsafeRunSync

testRoute(Login())
// res0: String = "Login page"
testRoute(Hello("world"))
// res1: String = "Hello, world"
testRoute(BlogPost("my-slug", 1))
// res2: String = "Blog post with id: 1, slug: my-slug"

You can also check that requests matching none of your routes are not handled by the service:

import org.http4s.{Method, Uri}

def unhandled(method: Method, path: String) =
  service
    .run(Request[IO](method = method, uri = Uri(path = path)))
    .value
    .unsafeRunSync

unhandled(GET, "/fake")
// res3: Option[org.http4s.Response[IO]] = None

// Not handled by `Hello` because the method doesn't match
unhandled(POST, "/hello/world")
// res4: Option[org.http4s.Response[IO]] = None
